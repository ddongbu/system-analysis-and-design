
    ▶ 프로토타이핑 기법의 장점
    ✓ 분석 및 설계에 사용자가 동참
    ✓ 말이나 추상적 형태가 아닌, 구체적인 형태로 요구사항을 포착함.

    ▶ 프로토타이핑 방법




   ◆ CASE 도구들
    = CASE(Computer-Aided Software Engineering)는 시스템 분석가들이 정보시스템 개발을 위해 사용하는 자동화된 소프트웨어 도구이다.
    = SDLC 전반에 걸쳐 사용할 수 있다.
    = CASE 도구 벤더들은 도구들 간에 정보공유가 보다 쉽게 이루어질 수 있도록 표준 데이터베이스와 데이터 변환 유틸리티들을 사용함으로써 자신들의 도구들을 ‘개방적인’ 것으로 만들어 놓았다.
    = 리파지토리라 불리는 통합적인 표준 데이터베이스는 다양한 제품들과 도구들을 통합시킬 수 있다.

    ▶ CASE 도구의 일반적인 유형들
    ✓ 다이어그래밍 도구
    ✓ 컴퓨터 화면 및 리포트 생성기
    ✓ 분석 도구
    ✓ 중앙 리파지토리 : 정보들을 통합적으로 저장할 수 있도록 해줌
    ✓ 문서 생성기
    ✓ 코드 생성기

    ▶ 리파지토리(repository)
    = 모든 다이어그램, 양식 및 리포트 정의, 데이터 구조, 데이터 정의, 프로세스 흐름과 논리, 그리고 여타 조직 및 시스템에 관한 정의들을 포함하는 중앙집중화된 데이터베이스
   ◆ 애자일 방법론들(Agile Methodologies)
    = 3가지의 중요한 공통된 원칙들을 가지고 있는데,  예단적인(predictive) 방법론이 아니라 적응적(adaptive) 방법론에 초점을 맞춤
    =  역할보다는 사람에 초점을 맞춤,  자기 적응적(self-adaptive) 프로세스이다.

     적응적 방법론 초점
    = 예단적인 방법론을 채택하는 것은 프로세스나 제품이 예측 가능한 경우 공학 기반의 방법론이 최선이었다는 경험에 근거한 것.
    = 오늘날 혼란스러운 비즈니스 환경에 적합하지 않음. 따라서 보다 적응력 높은 방법론들이 필요하게 됨.

     역할보다는 사람에 초점
    = 사람은 언제든지 교체될 수 있는 존재이므로 역할만이 중요하다고 보는 공학 기반의 기법에 대한 비판적 시각을 보여줌.
    = 애자일 접근방법에서의 사람은 단지 역할을 수행하는 존재가 아니라 개발 프로젝트에 관한 고유한 재능을 가진 존재로 여겨진다.

     자기 적응적 프로세스
    = 애자일 방법론들 자체는 소프트웨어에 대한 자기 적응적인 개발 프로세스를 촉진시키는 특성을 가지고 있다.
    = 이러한 방법론들을 활용하는 개발 팀도 자신이 수행하고 있는 프로젝트와 수행 정황에 맞도록 이러한 방법론들을 적응적으로 활용하려는 노력을 해야 한다.

    ◆ 전사적 지원 관리(ERP : Enterprise Resource Planning)의 특징

• 대규모 조직의 각기 다른 기능과 정보시스템을 연결해 상호간에 정보 교환이 수월해지고, 조직의 효율성과 경영성과를 증진시킬 수 있다.
• 기업과 하나의 통합된 기술 플랫폼 전반을 통해 통일된 데이터 표준들과 비즈니스 프로세스들을 가용함으로써 조직의 중앙 집중화를 지원한다.


   ◆ 전사적 지원 관리의 장점

• 분산되어 단절된 데이터 통합
• 데이터 및 시스템 공유
• 데이터베이스의 중앙 집중화
• 기업의 상이한 부문 또는 부서의 통합적 관리
• 생산시간 손실 최소화
• 시스템 상의 재고 정확도 개선
• 실시간 대처 가능
• 투명한 회계 상황 파악 가능
• 데이터 흐름을 추적, 오류 해결
