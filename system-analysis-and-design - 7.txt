
 ❍ 소프트웨어 공급원
   • 아웃소싱(outsourcing) = 외주
   - 조직의 정보시스템 애플리케이션 및 운영의 일부 또는 전체에 관한 책임을 외부 기업에게 이양하는 것.
   ex)
    = 아웃소싱 업체가 애플리케이션을 개발하고 자신의 컴퓨터에서 운영하는 것(급여 애플리케이션)
    = 아웃소싱 업체가 고객사 컴퓨터에 있는 애플리케이션을 직접 운영하는 것.

   ▶ 조직이 정보시스템의 운영을 아웃소싱하는 이유
    = 아웃소싱은 비용대비 효과가 높을 수 있다.
    = 매우 안정적인 컴퓨터 애필리케이션을 매우 낮은 비용으로 가동시킴으로써 규모의 경제를 달성할 수 있을 것이다.

   ▶ 조직이 컴퓨터 애플리케이션들에 대한 관리를 위해 자사의 정보처리부서 전체를 해체하면서까지 외부의 아웃소싱 서비스 업체를 활용하는 이유
    = 정보시스템 부서가 당면한 운영상의 문제점들을 극복하기 위해서이다.

   - 아웃소싱은 시스템 분석가가 꼭 알아야하는 대안이다.
   - 아웃소싱 여부는 시스템 분석 단계에서 다루어지는 것이 바람직하다.
   - 왜냐하면, 어떤 아웃소싱 업체가 조직의 니즈를 가장 잘 충족시켜줄 수 있는지 판단할 수 있기 때문이다.
   - 그러나 아웃소싱을 고려하지 않기로 했다면, 개발하고자 하는 시스템의 소프트웨어 구성요소들에 대해 직접 개발하는 대신 구매해서 사용할 수 있는지 고려해볼 필요가 있다.

   • 소프트웨어 공급원
   - 소프트웨어를 생산하는 조직은 다섯 가지의 주요 범주로 분류될 수 있다.
   - IT 서비스 업체, 패키지 소프트웨어 생산업체, 전사적 솔루션 소프트웨어 벤더, 오픈소스 소프트웨어, 자체 개발

   ◆ IT 서비스 업체
    = 다음과 같은 경우 활용된다.
    ✦ 자체적으로 시스템을 개발하는데 필요한 자원이 부족할 때
    ✦ 적합한 상용 솔루션을 찾을 수 없을 때
    = IT 서비스 업체들은 기업들이 자신의 용도에 맞게 맞춤화된 정보시스템을 개발할 수 있도록 도와준다.

    = 이러한 업체들은 고객사를 위해 애플리케이션을 개발하고, 호스팅해주며, 운영해준다.
    = IT 서비스 업체들은 정보시스템 및 특정 비즈니스 분야에 대해 경험과 지식을 갖고 있는 컨설턴트들을 채용한다.
    → 컨설턴트들은 기업에서 자체 개발할 때 사용하는 것과 동일한 다수의 방법론들, 기법들, 도구들을 사용한다.

   ◆ 패키지 소프트웨어 생산업체
    = 프리패키지(prepackaged) 시스템 또는 상용(off-the-shelf) 시스템이라 불리기도 하는 소프트웨어를 개발한다.
    = 상용 소프트웨어는 조직의 요구사항을 최대 70% 정도 만족시키는 것으로 추정된다.

    ▶ 소프트웨어 두 가지 유형
    ✓ 일괄(turnkey) 시스템
     = 사용자의 특정한 니즈에 맞춘 수정이 불가능한 소프트웨어

    ✓ 비일괄(non-turnkey) 시스템
     = 조직의 니즈에 보다 근접시키기를 원하는 사용자에 의해 수정되거나 확장되는 것이 가능하기도 한 소프트웨어
     = 즉, 커스텀마이징(수정)이 가능하다.

   ◆ 전사적 솔루션 소프트웨어(enterprise solution)
    = 전사적자원관리시스템(Enterprise Resource Planning system : ERP)이라 부르기도 한다.
    = 하나의 정보시스템 내에서 연속적으로 처리될 수 있도록 기존의 비즈니스 기능 각각을 일련의 모듈들로 통합한 시스템.
    = ERP 소프트웨어 솔루션은 일련의 연결된 모듈들로 구성되어 있다.
    = 각 모듈은 회계, 유통, 생산, 인적자원 등과 같은 전통적인 개별 기능들을 지원한다.
    = 모듈 방식을 취하는 접근방법은 전통적 접근방법과는 달리 비즈니스 기능 영역보다는 비즈니스 프로세스에 초점을 두고 통합되어 있다.
    → 기업은 ERP 시스템을 이용함으로써, 비즈니스 프로세스 전체를 하나의 정보시스템으로 통합할 수 있다.

    ▶ 전사적 솔루션 접근방법의 장점
    = 비즈니스 프로세스 전체와 관련된 데이터들을 한 곳에 통합시켜 놓은 데이터 저장소
    → 단일 저장소를 통해 데이터를 보다 일관성 있고 정확하게 유지할 수 있을 뿐만 아니라 유지보수 노력도 절감시킬 수 있다.
    = 필요에 따라 해당 모듈들을 추가해 나갈 수 있는 모듈의 유연성.
    → 추가하는 모듈은 기존 시스템에 바로 통합될 수 있다.
